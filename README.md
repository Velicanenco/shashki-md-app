# SHASHKI MD — рабочий прототип Mini App

Telegram Mini App для перегона авто из США в Молдову: калькулятор растаможки по молдавским тарифам, каталог реальных лотов с Copart, форма заявки (марка → серия, бюджет, телефон +373) и полуавтоматическая обработка заявок через Reply в Telegram. Оформление — тёмная тема с анимациями (прелоадер логотипа, скролл-ревил карточек, переключение RU/RO).

## Из чего состоит

- `backend/main.py` — сервер на FastAPI + aiogram 3. Отдаёт фронтенд, каталог машин и принимает заявки.
- `webapp/index.html` — сам Mini App: прелоадер с логотипом, калькулятор, каталог, форма заявки. Полностью на RU по умолчанию, с переключателем на RO (молдавский/румынский) в шапке.
- `cars_template.csv` — таблица с реальными лотами (3 BMW, которые вы прислали), которую заливаете в Google Sheets для боевого режима.

## Что нового по сравнению с прошлой версией

**Бренд.** Название сменено на SHASHKI MD, логотип пересобран в CSS/SVG (не растровым файлом) с анимацией "прогрузки" при открытии — так он не размывается на любом экране и грузится мгновенно, без лишнего файла.

**Языки.** Все тексты интерфейса переведены на румынский (молдавский государственный) и переключаются кнопкой RU/RO в шапке без перезагрузки страницы.

**Калькулятор — Молдова, а не Украина.** Полностью переписан на реальные молдавские ставки: акциз (accize) по объёму двигателя и возрасту авто, сбор за таможенное оформление ~0.4%, отдельно налог на "роскошь" при стоимости от 600 000 MDL, TVA 20% — но только если включить чекбокс "коммерческий ввоз / юр. лицо" (для физлиц, ввозящих авто для себя, по открытым источникам НДС не применяется). Важная честная оговорка: ставки взял из открытых калькуляторов молдавских брокеров (totalasig.md, rapidasig.md), они совпадают в контрольных точках (возраст 0-2 и 20+ лет), но полной таблицы по всем промежуточным возрастным группам в открытом доступе я не нашёл — для возраста между этими точками использована линейная интерполяция. Это помечено прямо в коде комментарием и в интерфейсе плашкой "ориентировочный расчёт". **Перед реальными продажами клиентам стоит проверить таблицу ставок с вашим таможенным брокером** — это буквально один объект в начале `<script>` в `index.html` (`PETROL_BRACKETS` / `DIESEL_BRACKETS`), поправить самостоятельно займёт 10 минут.

**Каталог.** Три лота, которые вы прислали (BMW 330i Sport 2017, BMW 320i 2015, BMW 320i 2016) — со всеми деталями: VIN, пробег, повреждения, локация, дата аукциона, сертификат, ссылка на сам лот Copart. По фото — честно: я не смог забрать присланные вами фотографии как файлы (в этой изолированной среде они были видны мне только "как картинка в переписке", без файла на диске, а Copart блокирует автоматическое скачивание фото по своим правилам). Вместо фото сейчас — стилизованная карточка-заглушка с ценником. Если пришлёте фото как обычные файлы-вложения (не вставкой в текст) или просто зальёте их куда угодно (Imgur, ваш же Google Диск с публичной ссылкой) — я в одну правку подключу их как `photo_url` в таблице, и они будут показываться в карточках.

**Заявка.** Телефон теперь с зафиксированным префиксом +373. Бюджет — те же пресеты плюс "Другое", где можно вписать сумму вручную. Добавлены марка авто и зависимая от неё серия/модель (для BMW — 1-8 серии, X1-X7, M- и i-линейки; для остальных 12 популярных марок — своя структура моделей, а не общий список).

**Оформление.** Вдохновлялся стилистикой darktrader.co (тёмная тема, крупная жирная типографика, контрастные акценты) — сделал похожее, но не копию: анимированные градиентные пятна на фоне, прелоадер логотипа, плавающий индикатор активной вкладки, скролл-ревил карточек, анимированное появление результата калькулятора.

## Как это работает (логика полуавтомата) — без изменений

1. Клиент открывает Mini App через кнопку у бота, заполняет заявку.
2. Сервер сверяет заявку с Google-таблицей машин.
3. Совпадение есть → бот сразу присылает клиенту варианты.
4. Совпадений нет → вам в Telegram приходит "Нова заявка №N" — отвечаете обычным **Reply** (текст/фото), бот сам пересылает нужному клиенту.

## Проверено локально

Как и в прошлый раз — не смог запустить вживую в Telegram из этой песочницы (нет доступа к api.telegram.org и pypi.org отсюда). Поднял локальный тестовый сервер и прогнал весь фронтенд через Playwright: прелоадер отрабатывает, калькулятор посчитан вручную и сверен построчно (акциз/сбор/TVA/итого), каталог рендерится, каскад марка→серия работает (проверено на BMW и Toyota), бюджет "Другое" появляется корректно, переключение RU→RO проверено на всех экранах, форма отправляется и получает ответ. `backend/main.py` проверен на синтаксис. Логотип-бейдж в шапке при первом прогоне вышел нечитаемым при 34px — переделал на компактный монограм "SH / MD", перепроверил скриншотом.

## Как развернуть (без изменений от прошлой версии)

1. Токен бота — @BotFather → `/newbot`.
2. Google Sheet с колонками из `cars_template.csv` → **Файл → Поделиться → Опубликовать в интернете → CSV** → это `CARS_SHEET_CSV_URL`.
3. Хостинг с HTTPS — Render.com (бесплатно): Build `pip install -r backend/requirements.txt`, Start `python backend/main.py`, переменные окружения из `backend/.env.example`.
4. Локально: `cd backend && pip install -r requirements.txt && cp .env.example .env` (заполнить) `&& python main.py`.
5. Тест в Telegram: `/start` у бота → "🚀 Открыть приложение".
